# 月別 入金・出金 集計アプリ

入金額と入金月が記載された Excel ファイルと、出金額と出金月が記載された Excel ファイルをアップロードして、月ごとの入金合計・出金合計・差額を一覧表示し、結果を Excel でダウンロードできる Streamlit アプリです。

---

## セットアップ方法

1. Python 3.10 以上をインストールしてください。
2. このフォルダに `cd` で移動し、依存ライブラリをインストールします。

```bash
pip install -r requirements.txt
```

---

## 起動方法

```bash
streamlit run app.py
```

ブラウザで `http://localhost:8501` が自動で開きます。

## ファイル形式の説明

### 想定する入金ファイル

- 例: `入金日１_2024年9月～2025年8月.xlsx`, `入金日２_2023年9月～2024年8月.xlsx`

### 想定する出金ファイル

- 例: `国内支払１_2024年9月～2025年8月.xlsx`, `国内支払２〜５_2023年9月～2024年8月.xlsx`

### Excel の特徴

- データは Sheet0 にある想定です。
- ヘッダーは7行目前後にあります（手動で変更できます）。
- 月の列には「9月 2025 ( 23 )」のような文字列が入っており、自動で `YYYY-MM` に正規化します。
- 同じ月の明細行では月列が空欄になることがあります。空欄の場合は直前の月を引き継いで集計します。
- 商談名が空欄の行は小計の可能性が高いため、原則として除外します。
- 「合計 / 小計 / 総計 / Grand Total / Total / 計」を含む行は初期状態で除外します。

### 自動判定するキーワード

| 種別 | キーワード |
|---|---|
| 月列 | 入金日, 支払日 |
| 金額列 | 税込, 合計, 原価総額, 入金額 |
| 商談名列 | 商談名, 案件名, 取引先, 件名, 摘要 |

`config.json` を編集する、または画面上で変更できます。

## 機能一覧

1. 入金 / 出金 Excel の複数アップロード
2. 自動列判定（月 / 金額 / 商談名）
3. データ確認モード（先頭30行プレビュー）
4. ヘッダー行の手動指定
5. 列名の手動指定（ファイルごと）
6. 期間フィルター（開始月・終了月）
7. 月別集計（入金合計 / 出金合計 / 差額）
8. 月別ドリルダウン（タブで入金 / 出金）
9. ファイル別集計（件数 / 除外件数 / 合計金額）
10. 除外行確認（理由・元データ）
11. 除外行の手動復活（チェックボックスで集計に戻す）
12. 処理ログ表示
13. Excel ダウンロード（6シート構成: 月別集計 / 入金明細 / 出金明細 / 除外行一覧 / ファイル別集計 / 設定・処理ログ）
14. 列指定・キーワードを `config.json` に保存 / 再読み込み

## 注意事項

- アップロードファイルはローカルメモリ上で処理され、外部送信はしません。
- 金額が文字列（`¥1,234`, `1,234円`, `(1,234)`, `▲1,234` など）でも数値として集計します。
- 月の判定は次の優先順で行います: `YYYY-MM` → 「M月 YYYY」 → pandas の汎用日付パース。
- 商談名が空欄かつ金額だけある行は「小計の可能性が高い」と判断して除外します。明らかに必要な場合は除外行リストでチェックを入れて集計に戻してください。
- 小計・合計の二重計上を防ぐことを最優先しています。

## よくあるエラー

| エラー | 原因 | 対処法 |
|---|---|---|
| `月列または金額列が判定できませんでした` | 自動判定のキーワードに該当する列名がない | 「列名の手動指定」セクションでファイルごとに月列・金額列を明示する |
| `Excel読み込み失敗: Worksheet ... is not in workbook` | シート構成が想定と異なる | 想定: 先頭シート (Sheet0)。`app.py` の `read_excel_with_header` を変更 |
| ヘッダーが正しく取れない | ヘッダー行番号がずれている | データ確認モードで実際のヘッダー行番号を指定し直す |
| Excel ダウンロードが空 | 期間フィルターで対象月が外れている / 全行除外されている | フィルターを広げる、除外行を復活する |

## Streamlit Community Cloud へのデプロイ

このリポジトリは Streamlit Community Cloud (https://share.streamlit.io) でそのままデプロイできます。

### 手順

1. https://share.streamlit.io にアクセスし、GitHub アカウント（`haracchi1202`）でサインインする。
2. **「Create app」 → 「Deploy a public app from GitHub」** または **「Deploy from existing repo」** を選択。
3. 以下を入力:
   - **Repository**: `haracchi1202/cashflow-monthly-aggregator`
   - **Branch**: `main`
   - **Main file path**: `app.py`
   - **App URL**: 任意のサブドメイン（例: `cashflow-haracchi`）
4. **「Deploy」** をクリック。初回は依存ライブラリのインストールに2〜3分かかります。
5. デプロイ完了後、`https://<chosen-subdomain>.streamlit.app` でアクセス可能。

### Private リポジトリの場合

Streamlit Community Cloud は Private GitHub リポジトリにも対応しています。初回サインイン時に **「Connect your GitHub account」** で `repo` スコープを許可してください。

### シークレット（任意）

将来 API キー等を扱う場合は、Streamlit Cloud の **「Settings → Secrets」** で TOML 形式で登録します。コード側は `st.secrets["KEY_NAME"]` で参照できます。本リポジトリの `.gitignore` は `.streamlit/secrets.toml` をコミット対象から除外済みです。

### 再デプロイ

`main` ブランチに push すると自動で再デプロイされます。
