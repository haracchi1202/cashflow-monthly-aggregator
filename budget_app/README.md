# 月別 入金・出金 集計 → 資金繰り予算 上書きアプリ

入金 / 出金 Excel をアップロードして月別に集計し、その結果を**資金繰り予算ファイル**
（例: `MBJ26年資金繰2026-04-23原追加.xlsx`）の **「確定入金 / 確定支払」の予算列**だけに
安全に上書きする Streamlit アプリです。

> 📌 既存の集計アプリ（`CFCREATOR/app.py`、Streamlit Cloud にデプロイ済み）はそのまま残し、
> こちらはローカル運用 / 別デプロイ用の独立アプリとして `CFCREATOR/budget_app/` 配下に
> 構築しています。デプロイ済みアプリの動作には影響しません。

---

## セットアップ

```bash
# Python 3.10+ を推奨
cd CFCREATOR/budget_app
pip install -r requirements.txt
```

## 起動

```bash
streamlit run app.py
```

ブラウザで `http://localhost:8501` が自動で開きます。

## 想定するファイル形式

### 入金ファイル
- 例: `入金日１_2024年9月～2025年8月.xlsx`
- ヘッダーは7行目前後（手動で変更可）
- 月列：「9月 2025 ( 23 )」「2025/09」「2025年9月」「令和7年9月」など混在 → 自動で `YYYY-MM` に正規化
- 同じ月の明細行で月列が空欄の場合は直前の月を引き継ぐ
- 商談名が空欄の行は小計の可能性が高いため除外

### 出金ファイル
- 例: `国内支払１_2024年9月～2025年8月.xlsx`
- フォーマットは入金ファイルと同じ

### 予算ファイル
- 例: `MBJ26年資金繰2026-04-23原追加.xlsx`
- 対象シート: `MBJ上半期`, `MBJ下半期`
- 対象行: `確定入金（カクテイニュウキン）`, `確定支払（カクテイシハライ）`
- 各月に「予算列」「実績列」が存在 → **予算列のみ**を上書き
- 月ヘッダは結合セルでも OK（例: `4月` がC〜D列に結合、その下に `予算 / 実績`）
- シート名に `26年` などが含まれる場合は会計年度として認識し、`4月` のような年なしの月表記でも `2026-04` などに自動推測

## 機能一覧

1. 入金 Excel アップロード（複数）
2. 出金 Excel アップロード（複数）
3. 予算 Excel アップロード（1ファイル）
4. データ確認モード（先頭30行プレビュー / ヘッダー行手動指定）
5. 自動列判定（月 / 金額 / 商談名）・列の手動指定
6. 月別集計（入金合計 / 出金合計 / 差額）
7. 期間フィルター（表示用）
8. 月別ドリルダウン（タブで入金 / 出金 / 件数 / 合計）
9. ファイル別集計
10. 除外行確認 / 手動復活（チェックボックスで集計に戻す）
11. 予算ファイル解析（シート別の検出結果表示）
12. 更新開始月 / 更新終了月の指定
13. 上書き前確認（上書き予定一覧 / 差額表示 / 更新理由）
14. 未照合月チェック（集計のみ / 予算のみ）
15. 上書き実行（チェックボックスで確認後にボタン有効化）
16. 上書き後比較（実測値を再読み込みして表示）
17. 更新済み予算 Excel のダウンロード
18. 比較レポート Excel ダウンロード（11シート構成）

## 比較レポート（11シート）

| シート | 内容 |
|---|---|
| 月別集計 | 入金合計 / 出金合計 / 差額 |
| 入金明細 | 集計対象の入金行 |
| 出金明細 | 集計対象の出金行 |
| 除外行一覧 | 理由・行番号・金額 |
| ファイル別集計 | ファイルごとの件数・合計・除外件数 |
| 上書き前データ | 予算ファイル内の (シート, 月, 区分, セル) と上書き前値 |
| 上書き後データ | 上書き後の予定値 + 更新対象フラグ |
| 上書き前後比較 | 上書き前/後・差額・更新対象・更新理由 |
| 更新対象一覧 | 更新対象のみを抜粋 |
| 未照合月一覧 | 集計のみ / 予算のみ の月 |
| 処理ログ | 設定・期間・各処理ログ・書き込み詳細 |

## 安全設計

- 原本ファイルは**絶対に変更しない**（メモリ上のコピーに対して上書き）
- 上書きするのは事前検出した予算列セル（確定入金/確定支払 × 各月の予算列）**のみ**
- **実績列**には絶対に書き込まない
- 他シート・他行・数式・書式は変更しない（openpyxl の `data_only=False` で数式・書式維持）
- 「指定期間外」「集計データなし」「月未一致」のセルは更新せず、その旨を更新理由として記録
- 上書き実行は確認チェックボックスをオンにした時のみ有効化
- 上書き後に対象セルを再読み込みして実測値と比較

---

## 月変換仕様

以下をすべて `YYYY-MM` に正規化します。

- `2025-09` / `2025/09` / `2025.09`
- `2025年9月` / `2025年09月`
- `9月 2025` / `9月 2025 ( 23 )`
- `令和7年9月`
- datetime / Timestamp 型のセル
- 予算ファイルの月のみ表記（`4月` 等）はシート名（`MBJ上半期 / 下半期`）と会計年度設定から推測

---

## 除外行ルール

初期状態で除外する行:
- 月が判定できない
- 金額が空欄
- 商談名が空欄
- 「合計 / 小計 / 総計 / Grand Total / Total / 計」を含む行

ユーザーが手動で「復活」可能（除外行一覧のチェックボックス）。

---

## 設定保存

`config.json` を画面から保存・再読み込み・ダウンロードできます。

主な項目:
- `default_header_row`
- `month_column_keywords` / `amount_column_keywords` / `deal_column_keywords`
- `exclude_keywords`
- `budget_target_sheets`（既定: `MBJ上半期`, `MBJ下半期`）
- `budget_fiscal_year`（既定: 2026）

## よくあるエラー

| エラー | 原因 | 対処法 |
|---|---|---|
| `月列または金額列が判定できませんでした` | 自動判定キーワードに該当列名がない | 「列名の手動指定」で月列・金額列を指定 |
| `『予算/実績』サブヘッダ行が見つかりません` | 予算ファイルの構造が想定と異なる | サイドバーの「対象シート」を確認、または該当シート名に変更 |
| `確定入金/確定支払 行を特定できません` | 行ラベル文言が「確定入金」「確定支払」を含まない | 元ファイルのラベルを確認。フリガナや行追加で文字列がずれていないか |
| 月のみ表記が認識されない | シート名に年度ヒントなし | `会計年度` 入力で年度を指定 |
| `Excel読み込み失敗: Worksheet ... is not in workbook` | 入金/出金 Excel のシート構成が想定と異なる | 先頭シートにデータがあるか確認 |
