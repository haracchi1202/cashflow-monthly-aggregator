# 資金繰り予算更新アプリ（確定 + 予測 フル機能版）

確定入金 / 確定支払 / 予測入金 / 予測支払 の 4 種類の Excel をアップロードして
月別に集計し、**選択した更新モード**（確定のみ / 予測のみ / 確定＋予測 合算）で
資金繰り予算ファイル（例: `MBJ26年資金繰2026-04-23原追加.xlsx`）の
**「確定入金 / 確定支払」の予算列**だけを安全に上書きする Streamlit アプリです。

> 📌 デプロイ済みの集計アプリ（`CFCREATOR/app.py`）および確定のみ版（`CFCREATOR/budget_app/`）には
> 一切影響を与えません。本アプリは `CFCREATOR/budget_app_full/` の独立アプリです。

---

## セットアップ

```bash
cd CFCREATOR/budget_app_full
pip install -r requirements.txt
```

## 起動

```bash
streamlit run app.py
```

ブラウザで `http://localhost:8501` が自動で開きます。

## 共通データ構造（将来 Zoho API 連携を見据えた設計）

すべての入力ファイルを以下の DataFrame に正規化します。
データ読み込み部分が `excel_reader.py` に分離されているため、
将来 Zoho CRM / Zoho Analytics API を追加する場合は、API 取得結果を
同じスキーマに変換するアダプタを追加するだけで集計以降のロジックを
そのまま再利用できます。

| 列 | 意味 |
|---|---|
| `source_file` | 元ファイル名（または API ソース識別子） |
| `source_group` | `確定入金` / `確定支払` / `予測入金` / `予測支払` |
| `transaction_status` | `confirmed` / `forecast` |
| `transaction_type` | `income` / `payment` |
| `target_month` | `YYYY-MM` |
| `transaction_date` | 元の月セルそのまま（参考用） |
| `amount` | 円（float） |
| `client_name` | 顧客名 |
| `deal_name` | 商談名 |
| `raw_row_index` | 元 Excel 上の行番号（1始まり） |

## 想定するファイル

### 確定入金ファイル
- `入金日１_2024年9月～2025年8月.xlsx`
- `入金日２_2023年9月～2024年8月.xlsx`

### 確定支払ファイル
- `国内支払１_2024年9月～2025年8月.xlsx` ほか

### 予測入金ファイル
- `予測売上2024年度初回入金月_複製済み.xlsx`
- `予測売上2024年度残金入金月_複製済み.xlsx`

### 予測支払ファイル
- `予測売上2024年度初回支払日_複製済み.xlsx`
- `予測売上2024年度残金支払日_複製済み.xlsx`
- `予測売上2024年度予備支払日_複製済み.xlsx`

### 資金繰り予算ファイル
- `MBJ26年資金繰2026-04-23原追加.xlsx`
- 対象シート: `MBJ上半期`, `MBJ下半期`
- 対象行: `確定入金（カクテイニュウキン）`, `確定支払（カクテイシハライ）`
- 各月に「予算列」「実績列」が存在 → **予算列のみ**を上書き
- 月ヘッダが結合セルでも OK、`4月` 等の年なし表記もシート名と会計年度から推測

## 機能一覧

1. 確定入金 / 確定支払 / 予測入金 / 予測支払 のアップロード（各複数可）
2. 予算 Excel アップロード（1 ファイル）
3. データ確認モード（先頭 30 行プレビュー / ヘッダー行手動指定）
4. 自動列判定 + 列の手動指定（月 / 金額 / 商談名 / 顧客名）
5. ファイル単位の追加除外キーワード設定
6. 月別集計（確定入金 / 確定支払 / 確定差額 / 予測入金 / 予測支払 / 予測差額 / 合算入金 / 合算支払 / 合算差額）
7. 期間フィルター（表示用）
8. 月別ドリルダウン（4 タブ: 確定入金 / 確定支払 / 予測入金 / 予測支払）
9. ファイル別集計（区分・確定/予測・件数・合計金額・除外件数）
10. 除外行確認 / 手動復活
11. 予算ファイル解析（シート別検出結果）
12. **更新モード選択**: 確定のみ / 予測のみ / 合算
13. 更新開始月 / 更新終了月の指定
14. 上書き前確認（更新対象セル一覧 / 差額 / 更新理由）
15. 未照合月チェック（集計のみ / 予算のみ）
16. 上書き実行（確認チェック後にボタン有効化）
17. 上書き後比較（実測値）
18. 更新済み予算 Excel のダウンロード
19. 比較レポート Excel ダウンロード（14 シート構成）

## 比較レポート（16 シート構成）

| シート | 内容 |
|---|---|
| 月別集計 | 確定 / 予測 / 合算 を 1 行に並べた全列一覧 |
| 月別集計_確定のみ | 確定入金 / 確定支払 / 確定差額 だけを抜粋 |
| 月別集計_予測のみ | 予測入金 / 予測支払 / 予測差額 だけを抜粋 |
| 確定入金明細 | 集計対象の確定入金行 |
| 確定支払明細 | 集計対象の確定支払行 |
| 予測入金明細 | 集計対象の予測入金行 |
| 予測支払明細 | 集計対象の予測支払行 |
| 確定＋予測月別集計 | 合算月別集計（確定＋予測 入金合計 / 支払合計 / 差額） |
| 除外行一覧 | 理由・行番号・金額・区分 |
| ファイル別集計 | ファイル単位の件数・合計・除外件数 |
| 上書き前データ | 予算ファイル内の (シート, 月, 区分, セル) と上書き前値 |
| 上書き後データ | 上書き後の予定値 + 更新対象フラグ |
| 上書き前後比較 | 上書き前/後・差額・更新対象・更新理由 |
| 更新対象一覧 | 更新対象のみ抜粋 |
| 未照合月一覧 | 集計のみ / 予算のみ |
| 処理ログ | 設定・期間・更新モード・各処理ログ・書き込み詳細 |

## 安全設計

- 元ファイルは**絶対に変更しない**（メモリ上のコピーに対して上書き）
- 上書きするのは予算列セル（確定入金/確定支払 × 各月）のみ
- **実績列**には絶対に書き込まない
- 他シート・他行・数式・書式は変更しない
- 「指定期間外」「集計データなし」「月未一致」のセルは更新せず、更新理由を記録
- 上書き実行は確認チェックボックスをオンにした時のみ有効化
- 上書き後に対象セルを再読み込みして実測値と比較

---

## 月変換仕様

以下をすべて `YYYY-MM` に正規化します。

- `2025-09` / `2025/09` / `2025.09`
- `2025年9月` / `2025年09月`
- `9月 2025` / `9月 2025 ( 23 )`
- `令和7年9月`
- datetime / Timestamp 型のセル
- 予算ファイルの月のみ表記（`4月` 等）はシート名（`MBJ上半期 / 下半期`）と会計年度設定から推測

---

## 除外行ルール

初期状態で除外する行:
- 月なし
- 金額なし
- 商談名空欄
- 「合計 / 小計 / 総計 / Grand Total / Total / 計」を含む行

ユーザーが手動で「復活」可能。

## 将来 Zoho API 連携への拡張ポイント

本アプリの読み込み部分（`excel_reader.parse_file`）は次のように分離されています。

```
[各種データソース] → [parse_file または 同等のアダプタ] → 共通 DataFrame
                                                          ↓
                                                  aggregator.build_dataframes
                                                          ↓
                                                  budget_updater + report_writer
```

Zoho 連携時は次のように追加することで集計以降のロジックをそのまま再利用できます。

- `zoho_reader.py`: Zoho OAuth / refresh token 管理 / API クライアント
- `zoho_reader.fetch_confirmed_income()`: Zoho Analytics の確定入金レポート → 共通 DataFrame
- `zoho_reader.fetch_forecast_income()`: Zoho CRM の予測売上 → 共通 DataFrame

各アダプタは `excel_reader.parse_file` と同じスキーマの DataFrame を返すだけで OK。

## よくあるエラー

| エラー | 原因 | 対処法 |
|---|---|---|
| `月列または金額列が判定できませんでした` | 自動判定キーワードに該当列名がない | 「列名の手動指定」で月列・金額列を指定。または config.json の `month_keywords_by_group` / `amount_keywords_by_group` を編集 |
| `『予算/実績』サブヘッダ行が見つかりません` | 予算ファイルの構造が想定と異なる | 「対象シート」を確認、または `予算/実績` の文字列が存在することを確認 |
| `確定入金/確定支払 行を特定できません` | ラベルが `確定入金` / `確定支払` を含まない | 元ファイルのラベルを確認 |
| 月のみ表記が認識されない | シート名に年度ヒントなし | 「会計年度」入力で年度を指定 |
